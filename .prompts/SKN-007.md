
## Problemas Conocidos ("Se ve raro a veces")

⚠️ **A identificar/pulir:**

1. **Transiciones MOVE ↔ MOVE_HARD (frames 6-17)**
   - Cuando el jugador cambia de "steering casual" a "steering held", ¿los frames tienen continuidad de silhueta?
   - ¿Hay "flip jitter" donde el cambio de asimetría es abrupto (p.ej., ala izquierda vs derecha)?
   - Compara frame 6 (move_calm col 0) vs frame 12 (move_hard col 0) — ¿se ve como mismo personaje pero esforzado, o como "otro sprite"?

2. **Row 6 Especiales (Dizzy, Trophy, Crown)**
   - ¿Frame 36 (dizzy) tiene postura coherente con el idle (frame 0)?
   - ¿Frame 38-39 (trophy/crown) usan cabeza correcta o hay asimetría pixel?
   - ¿Espressión facial es legible o pixeles mal colocados?

3. **BOOST (Row 3, frames 18-23) — Sparkle timing**
   - ¿Las 6 variantes de sparkle/glow frente al cuerpo del personaje tienen silhueta consistente?
   - ¿O hay frames donde los sparkles "no alinean" con la pose subyacente (brazos, cola, posición)?

4. **Animations loops (cíclicos: HOVER, MOVE, MOVE_HARD, BOOST)**
   - ¿La última frame empalma suavemente con la primera sin "brinco" o cambio de tamaño?
   - ¿O hay asimetría izquierda/derecha que viola la continuidad?

5. **Especial COMBO (Row 5, frames 30-35)**
   - Este branch ya mejoró el combo (usando CHEER smooth en lugar de badge flip).
   - ¿Las 6 variantes (x2, x3, x5, x10, x20) mantienen la misma postura base con badge diferente?

## Pasos de Trabajo

### 1. Inspeccionar el sprite en master (actual)

```bash
npm run dev  # or npm run preview
# Navega a http://localhost:5173 (web build)
```

En el navegador:
- Abre DevTools (F12) → Phaser inspector
- Juega algunos runs para ver cada animación completa (HOVER, MOVE, MOVE_HARD, BOOST, CHEER, SPECIAL)
- **Toma screenshots** de:
  - Frame 6, 12 (MOVE vs MOVE_HARD inicio)
  - Frame 11, 17 (MOVE vs MOVE_HARD final)
  - Frame 36, 38, 39 (Dizzy, Trophy, Crown)
  - Frame 18-23 (BOOST) — verifica alineación de sparkles
  - Frames 30-35 (COMBO) — verifica continuidad de badge

### 2. Comparar con branch `looking-for-character-sprite-HEAD-50-good`

```bash
git stash  # Si tienes cambios sin commit
git checkout looking-for-character-sprite-HEAD-50-good
npm run dev
# Inspecciona los mismos frames. ¿Se ven mejor? ¿Más consistentes?
```

Toma screenshots de los mismos frames. Compara lado a lado:
- ¿La nube de pensamiento (frame 37) ya está gone?
- ¿COMBO ahora sigue la pose CHEER en lugar de flipear badge?
- ¿Hay otros detalles de pixel que mejoraron?

### 3. Identificar frames problemáticos específicas

De los screenshots, lista:
- **Frame X — Problema:** (p.ej., "Frame 12 tiene ala derecha más pequeña que frame 6, parece inconsistencia")
- **Impacto:** (visual jitter, breaks immersion, confusing gameplay)
- **Fix:** (ajustar pixels, redimesionar ala, etc.)

### 4. Editar sprite en imagen editor

Dependiendo de cómo esté guardado el sprite:

**Opción A: Si tienes .psd o archivo fuente separado por frame**
- Abre el editor (Krita, Photoshop)
- Ubica el frame problemático
- Ajusta pixels (simetría, tamaño, alineación)
- Exporta frame actualizado

**Opción B: Si el sprite es un atlas monolítico**
- Usa herramienta de sprite editor (Aseprite, LibreSprite) o Python PIL
- Extrae el frame de la grid, edita, reinclusta en el grid
- Script: `scripts/repack-character.py` puede regenerar si tienes frames individuales

### 5. Validar en web + commit

Una vez editados frames:
```bash
npm run dev
# Inspecciona nuevamente
# ¿Se ve más consistente? ¿Animation loops suave?
```

Si está bien, commit:
```bash
git add public/assets/character.png src/config/CharacterSprite.ts (si cambió)
git commit -m "SKN-007: polish character sprite frames (MOVE/MOVE_HARD/COMBO transitions)

- Fixed asymmetry in frames X, Y, Z (description)
- Ensured smooth animation loop transitions
- Verified consistency with best frame from looking-for-character-sprite branch"
```

### 6. Merge de cambios del branch si es necesario

Si el branch `looking-for-character-sprite-HEAD-50-good` tiene arreglos que quieres adoptar:

```bash
# Opción: Cherry-pick commits específicos
git cherry-pick 7e4f430  # Combo smoothness + sad cloud removal
git cherry-pick e2b0dad  # Skin sad frame cleanup

# O merge completo del branch (cuidado: tiene WIP assets)
git merge looking-for-character-sprite-HEAD-50-good --no-commit
# Revisa cambios, limpia WIP, commit selectivamente
```
